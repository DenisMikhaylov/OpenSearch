

### Шаг 1: Установка Python

```bash
sudo apt update && sudo apt upgrade -y
sudo apt install python3 python3-venv python3-dev python3-pip -y
```

Проверьте успешность установки:

```bash
python3 --version
```
Вы должны увидеть версию, например `Python 3.13.5`.

### Шаг 2: Установка pip


Правильные способы работы с pip на Debian 13:

1.  **Виртуальное окружение (venv) — для проектов:** Это изолированная среда, в которой вы можете устанавливать любые пакеты без влияния на систему. Это **рекомендуемый** подход для разработки.
2.  **pipx — для глобальных CLI-инструментов:** Если вам нужно установить утилиту командной строки (например, `ansible`, `httpie`), используйте `pipx`. Он автоматически создает для каждого инструмента свое виртуальное окружение, но делает его команды доступными глобально.
3.  **apt — для системных пакетов:** Для библиотек, используемых всей системой, предпочтительнее использовать системный менеджер пакетов: `sudo apt install python3-<имя_пакета>`.


### Шаг 3: Создание и использование виртуального окружения

Это основной рабочий процесс для разработки на Python.

1.  **Создайте окружение** в папке вашего проекта:
    ```bash
    python3 -m venv ~/my_project/venv
    ```

2.  **Активируйте его:**
    ```bash
    source ~/my_project/venv/bin/activate
    ```
    После активации в начале строки приглашения появится `(venv)`.

3.  **Установите пакеты:** Теперь внутри активного виртуального окружения вы можете использовать `pip` без `sudo` и без ошибок:
    ```bash
    pip install requests flask
    ```

4.  **Завершите работу:**
    ```bash
    deactivate
    ```


### Шаг 5: Установка пакетов через apt (системные)

Если какая-то библиотека нужна всей системе, установите ее через `apt`. Названия пакетов обычно начинаются с `python3-`.
```bash
sudo apt install python3-numpy python3-requests
```

Этот подход гарантирует, что библиотеки будут обновляться вместе с системой и не вызовут конфликтов.

### 💎 Итог

Таким образом, `pip` на Debian 13 устанавливается вместе с Python через `apt`, но его использование для глобальной установки пакетов **блокируется** на уровне системы. Для работы с Python-проектами всегда создавайте **виртуальные окружения (venv)**, а для глобальных утилит используйте **pipx**. Это современная и безопасная практика, принятая в актуальных версиях Debian.
